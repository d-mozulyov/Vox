## Языковая политика проекта

### Русский язык (для команды разработки)

Следующие материалы должны быть **на русском языке**:

- Все файлы в директории `.kiro/` (steering, specs, конфигурации)
- Внутренняя документация для команды
- Коммуникация с агентом Kiro
- Комментарии в issue и pull request на русском (если участники русскоязычные)

**Обоснование**: Команда разработки русскоязычная, работа ведётся в Kiro на русском языке.

### Английский язык (для международной аудитории)

Следующие материалы должны быть **на английском языке**:

- Комментарии в коде
- Строковые константы в коде (логи, сообщения об ошибках, UI-тексты)
- README.md и другие публичные документы
- Документация в директории `docs/` (если появится)
- Названия переменных, функций, типов, пакетов
- Commit messages
- Issue и pull request titles/descriptions (для внешних контрибьюторов)

**Обоснование**: Проект публикуется на GitHub под лицензией MIT для международного сообщества. Разработчики по всему миру должны иметь возможность использовать, форкать и контрибьютить в проект.

### Практические правила

- При написании кода всегда используй английские названия и комментарии
- При работе со steering/specs всегда используй русский язык
- Если нужно перевести концепцию с русского на английский для кода — обсуди варианты перевода
- Документация API должна быть на английском
- Внутренние заметки и TODO в коде могут быть на русском, но перед коммитом должны быть переведены
