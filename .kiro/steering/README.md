# Steering Files - Руководство для агента Kiro

Этот каталог содержит steering файлы, которые определяют правила и контекст для работы агента Kiro над проектом Vox.

## Список steering файлов

1. **project-overview.md** - Общий обзор проекта Vox, его цели и особенности
2. **development-philosophy.md** - Философия разработки (KISS + Бритва Оккама + YAGNI)
3. **language-policy.md** - Языковая политика (русский для команды, английский для кода)
4. **technical-stack.md** - Технический стек и инфраструктура
5. **git-policy.md** - Политика работы с Git и GitHub

## Как это работает

Все steering файлы автоматически загружаются в контекст агента Kiro при работе с проектом. Это позволяет агенту:

- Понимать специфику проекта
- Следовать установленным правилам разработки
- Использовать правильные языки для разных типов документов
- Соблюдать технические требования

## Обновление steering файлов

Steering файлы должны обновляться по мере развития проекта. Если появляются новые правила, стандарты или изменяется архитектура - обновите соответствующие файлы.

## Следующие шаги

После завершения работы со steering файлами, следующий этап - создание specs (спецификаций) для конкретных задач разработки.
